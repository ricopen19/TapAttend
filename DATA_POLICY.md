# データの取り扱いについて

TapAttend で入力されたデータ（生徒名、出席状況など）がどのように扱われるかを説明します。

## データの保存場所

入力されたデータは、**利用者のブラウザ内（IndexedDB）にのみ保存**されます。

- 外部のサーバーやクラウドにデータが送信されることはありません
- データはブラウザごと・端末ごとに独立しています（iPad で入力したデータは PC には自動で反映されません）
- 別の端末にデータを移す場合は、JSON バックアップ機能を使って手動で行います

## 通信について

- **初回アクセス時**: GitHub Pages からアプリ本体（HTML/CSS/JavaScript）をダウンロードします
- **2回目以降**: Service Worker によりキャッシュされたアプリを使用するため、オフラインでも動作します
- **入力データに関する通信は一切発生しません**

## ホスティングサービスのログ

本アプリは GitHub Pages 上でホストされています。GitHub Pages では以下の情報が記録される場合があります：

- アクセス元の IP アドレス
- アクセス日時
- ブラウザの種類（User-Agent）

これらは GitHub のプライバシーポリシーに基づいて管理されます。
**出席データや生徒の個人情報はこのログに含まれません。**

## データの消失リスク

以下の操作を行うと、保存されたデータが消失します：

- ブラウザのキャッシュ / サイトデータの削除
- ブラウザの再インストール
- 端末の初期化

**定期的に JSON バックアップを取ることを強く推奨します。**

## データの管理責任

- データの管理（バックアップ、削除、端末の管理）は利用者自身の責任で行ってください
- 本アプリの開発者はデータにアクセスする手段を持っていません

## 個人情報の取り扱いに関する補足

生徒の個人情報の取り扱いが気になる場合、以下の運用を推奨します：

- **イニシャルで登録する**（例：「Y・T」「S・H」）
- **出席番号のみで登録する**（例：「1番」「2番」）
- **端末のパスコードロックを設定する**
- **共有端末での利用を避ける**

## まとめ

| 項目 | 内容 |
|---|---|
| データ保存先 | ブラウザ内（IndexedDB）のみ |
| サーバーへのデータ送信 | なし |
| オフライン動作 | 対応 |
| データの暗号化 | なし（ブラウザの標準的なストレージ） |
| アクセスログ | GitHub Pages の標準ログ（入力データは含まない） |
| バックアップ | 手動（JSON ファイル） |
| データ管理責任 | 利用者 |
